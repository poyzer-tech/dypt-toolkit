# dypt Task Workflow Rules

This reference will document the rules agents should follow when using dypt as the work tracker.

Rules to cover:

- Search before creating tasks.
- Read notes, children, and dependencies before implementation.
- Keep task notes current with durable context.
- Use dypt task links: `[#123](#task-123)`.
- Do not set leaf tasks to `in progress`.
- Preview cleanup before apply.
- Do not create markdown TODOs as a second tracker.
- Track follow-up work as dypt tasks.
